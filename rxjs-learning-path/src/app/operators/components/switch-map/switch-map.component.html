Operador switchMap

El operador switchMap toma cada valor emitido por el observable fuente y lo transforma en un nuevo observable.

Cancelación de Emisiones Anteriores: Si el observable fuente emite un nuevo valor antes de que el observable interno anterior complete, switchMap cancela la suscripción al observable interno anterior y se suscribe al nuevo observable.

Uso Común: Es ideal para manejar flujos de datos donde solo te interesa la última emisión, como en búsquedas en tiempo real o solicitudes HTTP dependientes.
